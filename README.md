# دانش‌یار — پلتفرم آموزش آنلاین (نسخه نمونه / MVP)

یک پلتفرم یادگیری آنلاین فارسی‌زبان (LMS) با چیدمان کامل راست‌به‌چپ (RTL)، ساخته‌شده با
**Next.js 14 (App Router)**، **TypeScript** و **Tailwind CSS**. این نسخه صرفاً با
**داده‌های نمونه (Mock)** کار می‌کند و نیازی به بک‌اند یا پایگاه‌داده ندارد.

پلتفرم دو نوع کاربر دارد: **دانشجو** و **استاد**، هرکدام با پنل اختصاصی خود.

## ⚙️ راه‌اندازی / Setup

```bash
npm install
npm run dev
```

سپس مرورگر را روی [http://localhost:3000](http://localhost:3000) باز کنید.

> ساخت نسخهٔ تولید: `npm run build && npm start`

## ✨ امکانات / Features

- **صفحهٔ فرود:** هیرو، دسته‌بندی‌ها، دوره‌های منتخب، آمار، مراحل کار، نظرات.
- **احراز هویت نمونه:** ورود/ثبت‌نام مبتنی بر `localStorage` + React Context، مسیریابی محافظت‌شده بر اساس نقش.
- **دوره‌ها:** فهرست با جست‌وجو و فیلتر (دسته‌بندی، سطح، رایگان/پولی) + صفحهٔ جزئیات با سرفصل آکاردئونی، نظرات و باکس ثبت‌نام/خرید.
- **پنل دانشجو:** داشبورد، دوره‌های من، پخش‌کنندهٔ دوره، کلاس‌های زنده، پروفایل.
- **پنل استاد:** داشبورد آماری، مدیریت دوره (ایجاد/ویرایش/حذف نمونه)، مدیریت دانشجویان، تحلیل‌ها (نمودارهای SVG)، پیام‌ها، کلاس‌های زنده.
- **پخش زنده:** فهرست جلسات با لینک «گوگل میت» (نمونه).
- **درباره ما:** تیم، ماموریت، ارزش‌ها، تماس.

## 🧱 پشتهٔ فنی / Tech Stack

- Next.js 14 (App Router) · TypeScript
- Tailwind CSS (با توکن‌های سفارشی رنگ/سایه)

## 📝 یادداشت‌های پیاده‌سازی / Implementation Notes

این موارد آگاهانه و برای سادگی و اجرای آفلاین انتخاب شده‌اند:

1. **ساختار مسیرها:** در سند اولیه، پنل‌ها به‌صورت گروه‌های مسیر `(student)` و `(instructor)` تعریف شده بودند.
   چون گروه‌های مسیر در Next.js هیچ بخشی به URL اضافه نمی‌کنند، هر دو روی `/courses` و `/dashboard`
   **تداخل** پیدا می‌کردند و باعث خطای ساخت می‌شدند. به‌جای آن از بخش‌های مسیر واقعی
   `/student/...` و `/instructor/...` استفاده شد. گروه `(auth)` بدون تداخل حفظ شد.

2. **فونت:** Vazirmatn از طریق `@import` در `globals.css` از CDN (jsDelivr) بارگذاری می‌شود
   تا بدون مرحلهٔ نصب اضافی، در مرورگر کاربر در دسترس باشد. برای محیط بدون اینترنت می‌توانید فونت را به‌صورت محلی میزبانی کنید.

3. **تصاویر:** به‌جای `next/image` با منابع راه‌دور، تصاویر شاخص و آواتارها با
   **گرادیان CSS + ایموجی/حروف اول نام** ساخته شده‌اند تا پروژه کاملاً آفلاین و بدون پیکربندی دامنه اجرا شود.

4. **داده‌ها:** همهٔ داده‌ها در `lib/mock/` هستند. عملیات‌هایی مثل «ذخیره پروفایل»، «ساخت دوره»،
   «ارسال پیام» و «حذف دوره» صرفاً نمایشی‌اند و در حافظهٔ نشست باقی می‌مانند (بدون ماندگاری واقعی).

5. **لینک‌های گوگل میت:** آدرس‌های `meet.google.com` نمونه و غیرواقعی هستند.
